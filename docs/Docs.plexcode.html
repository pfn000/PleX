Getting started

Give new developers a zero-conf path from install to first successful run.

  1. Install the PlexCode CLI and runtime for your platform.
  2. Create a project with plex init my-app.
  3. Run plex dev to launch the local environment.
  4. Ship with plex deploy after tests and policy checks pass.

Tooling and workflows

Make CLI workflows easy to scan and easy to use on phones.

Command Purpose Notes
plex init Create a new app, package, or library skeleton. Use templates for API, web, and runtime modules.
plex test Run unit, integration, and snapshot suites. Integrate coverage and CI-friendly output.
plex fmt Apply official formatting rules across the workspace. Keeps style consistent for contributors.
plex deploy Package, validate, and release build artifacts. Include environment policy and permission checks.
