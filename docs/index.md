PlexCode documentation now lives directly inside the repository's `docs/` folder and is rendered as part of this website.

## What this site covers

- GitHub Pages publishing from `main` + `/docs`
- docs pages written in markdown and rendered inside the site layout
- repository policies and setup guidance without forcing readers to jump out to GitHub views
- preparation work for upstreaming PleX file types to GitHub Linguist

## Recommended reading order

1. Start with [Getting Started]({{ '/getting-started.html' | relative_url }}).
2. Review [Site Setup]({{ '/site-setup.html' | relative_url }}) for the Pages configuration.
3. Open the [GitHub Linguist Guide]({{ '/github-linguist-upstream-custom-language.html' | relative_url }}) for the upstream language-definition plan.
4. Check the [Security Policy]({{ '/security.html' | relative_url }}) and [Legal]({{ '/legal.html' | relative_url }}) pages for repository policy information.
